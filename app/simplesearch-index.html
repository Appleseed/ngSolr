Simple Document Search

Simple Document Search is a Javascript search tool that helps you quickly and easily incorporate Apache Solr search into your new or existing web site. The following provides instructions on how to incorporate this functionality.


Add a Search Box to Your Web Pages

Users enter keyword searches on your web site through a search box. The box can appear on one or more pages, and can optionally provide search hints as the user types.

  1. Make sure that your web page is HTML5 compatible. The top of the HTML should look something like this: 
    <!DOCTYPE html>
<html lang="en">
    ...
  2. Copy the simplesearch.js file into your project folder
  3. Link the simplesearch.js file into your HTML page by including the following line in the HEAD of the page: 
    <!-- simple search -->
<script type="text/javascript" src="path/to/simplesearch.js"></script>
  4. Add the search box HTML inside of your page. 
    <!-- Search Box: the search box can be
    installed on any web page, including -->

<form ng-controller="SearchBoxController" ng-init="init()" ng-submit="submit()" ng-cloak>
    <div id="search-box">
        <input id="search-box-input" searchhints autocomplete="off"
            name="q" ng-model="userquery"
            placeholder='Enter placename or keywords to search for'
            type='text' ui-items="getHints(userquery)"
            />
        <button class="btn">Search</button>
    </div>
</form>
  5. Add the search app to the HTML tag at the top of the page. 
    <!DOCTYPE html>
<html lang="en" ng-app="simplesearch">
    ...

Create a Search Results Page

Copy the simplesearch.html page called search.html. 

<!-- search results -->
    <div id="result-panel" ng-controller="DocumentSearchResultsController" ng-init="init();">

    <!-- Document Listing: This is the list of documents that match
    our search query. Each document is listed inside of an 'article'
    block, and each presented property is displayed inside the {{ x }}
    braces. You can modify the formatting of the article block as you
    desire. However, if you want to change the attributes that are
    displayed, then you need to talk to the network administrator. -->
    <div id="search-results" ng-cloak>
    <div id="summary">Search Results ({{totalResults}} found)</div>
    <article ng-repeat="doc in documents" class="document">
    <a ng-href="{{doc.presentation_url}}" class="title">{{doc.title}}</a>
    <div class="thumb" ng-show="doc.dobj_proxy_small" style="margin-left:10px;">
    <a ng-href="{{doc.presentation_url}}" class="thumbnail">
    <img ng-src="{{doc.dobj_proxy_small}}" alt="{{doc.title}}" height="64px" width="64px" />
    </a>
    </div>
    <div class="text">{{doc.abstract}}</div>
    <div class="meta inline">
    <span class="region" ng-show="doc.region">{{doc.region}}</span>
    <span class="country" ng-show="doc.country">{{doc.country}}</span>
    <div class="existdates">
    <span>{{doc.fromDate}}</span>
    <span ng-show="doc.toDate">–</span>
    <span>{{doc.toDate}}</span>
    </div>
    </div>
    </article>
    </div>
    <!-- /document listing -->

    <!-- Pagination: If there are one or more pages of search results
    then we show a page listing here. The listing currently assumes that you
    are using the Twitter Bootstrap CSS template to get the 'pagination' class
    style here. If you aren't using Twitter Bootstrap then you can create your
    own style for the pagination links. -->
    <div class="search-results-pager pagination" ng-hide="error" ng-cloak>
    <ul>
    <li ng-repeat="page in pages" ng-class="{active:page.isActive,disabled:page.isDisabled}">
    <a id="page-{{page.number}}" ng-click="$parent.setPage(page.number)">{{page.name}}</a>
    </li>
    </ul>
    </div>
    <!-- /pagination -->

    </div>
    <!-- /search results -->
